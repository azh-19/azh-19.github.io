GIT TOPIC

Git merupakan salah satu sistem pengontrol versi (Version Control System) pada proyek software yang dibuat oleh Linus Torvalds. Git ini akan menjadi penghubung antara penyimpanan lokal (pc/komputer) ke penyimpanan di git yang nantinya orang lain bisa mengakses file yang kita upload dan bisa melihat juga update an filenya. Git dikenal juga dengan distributed revision control (VCS terdistribusi), artinya penyimpanan database Git tidak hanya berada dalam satu tempat saja. Semua orang yang terlibat dalam pengkodean proyek akan menyimpan database Git, sehingga akan memudahkan dalam mengelola proyek baik online maupun offline.

GITHUB

Selain git kita juga akan membutuhkan GitHub dimana GitHub ini yang akan mengnyimpan file yang di upload git dan orang lain bisa mengakses filenya di GitHub ini. GitHub ini adalah platfrom khusus developer yang dibuat karena terinspirasi dari cara bekerja para programmer. Gampangnya, GitHub merupakan manajemen project, sistem versioning code, sekeligus platfrom jaringan sosial bagi para developer seluruh dunia.

BASIC GIT COMMAND

- git init : Perintah ini digunakan untuk membuat repository baru/kosong

- git status : Perintah git status menampilkan daftra file yang berubah bersama dengan file yang ingin ditambahkan atau di-commit

- git log : Perintah ini akan menampilkan daftar commits yang ada di branch beserta detail-nya

- git add : Perintah git add bisa digunakan untuk menambahkan file ke index

- git commit : Perintah git commit digunakan untuk melakukan commit pada perubahan ke head

- git push : Push akan mengirimkan perubahan ke master branch dari remote repository yang berhubungan dengan direktori kerja anda

- git config : git config digunakan untuk mengatur konfigurasi tertentu sesuai keinginan penggunan, seperti email, username, format file, dll

- git checkout : perintah git checkout bisa digunakan untuk membuat branch atau untuk berpindah diantaranya

- git branch : perintah git branch bisa digunakan untuk me-list, membuat atau menghapus branch. Untuk menampilkan semua branch yang ada di repository

- git pull : Untuk menggabungkan semua perubahan yang ada di remote repository ke direktori lokal

- git clone : Perintah git clone digunakan untuk checkout repositori. Jika repositori berada di remove server

- git rm : Gunakan perintah ini untuk menghapus file dari index dan direktori kerja

- git show : Untuk menampilkan informasi tentang obejct pada git

- git fetch : Perintah ini digunakan untuk menampilkan semua object dari remoter repository yang tidak berada di direktori kerja lokal
